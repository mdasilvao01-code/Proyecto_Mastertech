# Sistema de Gestión de Incidencias - MASTERTECH

Sistema sencillo de gestión de incidencias técnicas usando archivos JSON para almacenamiento de datos.

## 📋 Requisitos

- Python 3.8 o superior
- No requiere librerías adicionales (usa solo librerías estándar de Python)

## 🚀 Instalación y Ejecución

1. Descarga el archivo `sistema_incidencias.py`

2. Ejecuta el programa:
```bash
python sistema_incidencias.py
```

## 📂 Archivos Generados

El sistema crea automáticamente 3 archivos JSON:

- `usuarios.json` - Almacena usuarios del sistema
- `incidencias.json` - Almacena las incidencias
- `comentarios.json` - Almacena comentarios de incidencias

## ✨ Funcionalidades por Rol

### 👨‍💼 Cliente
- ✅ Crear nuevas incidencias
- ✅ Ver sus incidencias
- ✅ Ver detalle de incidencias
- ✅ Añadir comentarios

### 🔧 Técnico
- ✅ Ver incidencias asignadas
- ✅ Ver todas las incidencias
- ✅ Cambiar estado de incidencias
- ✅ Añadir comentarios

### 👑 Administrador
- ✅ Dashboard con estadísticas
- ✅ Ver todas las incidencias
- ✅ Asignar técnicos
- ✅ Cambiar estados
- ✅ Ver usuarios del sistema

## 📊 Estados de Incidencia

- **Abierto** - Recién creada, sin asignar
- **En Proceso** - Asignada a técnico
- **Pendiente Cliente** - Esperando respuesta
- **Resuelto** - Problema solucionado
- **Cerrado** - Incidencia finalizada

## 🎯 Niveles de Prioridad

- **Crítica** - Bloquea completamente el trabajo
- **Alta** - Afecta significativamente
- **Media** - Afecta parcialmente
- **Baja** - No es urgente

## 💡 Ejemplo de Uso

1. Inicia sesión como cliente
2. Crea una nueva incidencia
3. Cierra sesión
4. Inicia sesión como administrador
5. Asigna la incidencia a un técnico
6. Cierra sesión
7. Inicia sesión como técnico
8. Cambia el estado a "En Proceso"
9. Añade comentarios
10. Cambia el estado a "Resuelto"

## 🛠️ Características Técnicas

- Interfaz de consola simple e intuitiva
- Almacenamiento en archivos JSON
- Sistema de autenticación básico
- Gestión de roles (RBAC)
- Timestamps para auditoría
- Sin dependencias externas

## 📝 Notas

- Los datos se guardan en archivos JSON en el mismo directorio
- Las contraseñas están almacenadas en texto plano (solo para demostración)
- El sistema es completamente funcional sin necesidad de base de datos
- Ideal para entornos de aprendizaje y pruebas
